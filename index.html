The thesis

Personal AI lives in classical mechanics. Enterprise AI lives in quantum mechanics.

Classical · Personal AI

You can enumerate every job.

You can predict any single call.

Guards are deterministic hooks.

State is traceable end to end.

Failure is one bot, one fix.

Quantum · Enterprise AI

You cannot enumerate every call.

You cannot predict any one of them.

Guards are sampled review.

State is aggregate metrics.

Failure is drift across a population.

The mistake people make: treating enterprise compliance as personal compliance with bigger budgets. It isn't.

Personal AI · The threat model

Your AI has the keys.

It reads your inbox. It can send messages, move money, touch code, edit your second brain, and commit to your repos. Anything you can do from the keyboard, the agent can do at three in the morning while you sleep.

A personal AI without compliance is an intern with admin access and no boss in the room.

Outbound is the highest blast radius. One pre-Bash hook scans every command for nine outbound patterns. Resend, GoHighLevel, Slack API, curl to email APIs, Twilio, WhatsApp, Gmail CLI, SMS. The hook forces the recipient, subject, and full body into the agent's working context. It never blocks. Four internal alert channels are exempt.

The agent runs with my credentials around the clock. Outbound gets the loudest hook because one bad email costs real money. Or relationships, which are worse.
Personal AI · The reasoning loop

The agent runs a seven-phase loop on any non-trivial task.

The loop is auditable. Each phase emits a structured artifact, so when something goes sideways I can trace which phase the agent skipped or got wrong.

  • 01ObserveEcho the intent in one sentence. Preflight gates. Reproduce before touching code.
  • 02ThinkPremortem. Surface the riskiest assumptions and check them.
  • 03PlanScope, deliverables, parallel work, feature breakdown.
  • 04BuildInvoke the capabilities the plan called out. Root cause at ingestion.
  • 05ExecutePer-criterion inline verification. Tool probes at every transition.
  • 06VerifyLive probe of user-facing artifacts. Re-read the original request.
  • 07LearnConjecture, refutation, changelog entry. Doc sync.
I call this the JAX Algorithm v6.3.0. Don't get hung up on the name; the point is the audit trail. The loop is what makes the agent auditable rather than just clever.
Personal AI · Action trust

Twenty-six command patterns sorted into three trust levels.

Every shell command passes through a pre-tool-use hook that classifies it before the agent runs it. The hook never blocks; it injects context the agent has to read.

Low
Read-only, local, harmless. Silent passthrough.
Medium
Reversible state changes. Allow with a warning. (curl, git commit, bun install.)
High
Hard to reverse, or shared production state. Strong confirmation context injected. (git push, rm -rf, sudo, SQL DROP.)

Example. The agent tries git push origin main --force. Classifier returns HIGH. The hook injects: "this rewrites remote history. Check branch protection." The agent now has to argue around that context to proceed. It rarely does.

The local-network exemption matters: curls to localhost get downgraded so dev loops don't become a warning storm.
Personal AI · Mode progression

Every bot ships with a five-mode dial. No bot starts at the top.

  • 01offKilled. The cron fires, the script exits before doing work.
  • 02dryDrafts run, nothing posts. Logs only.
  • 03reviewDrafts post to Slack for me to approve one at a time.
  • 04testPosts to a sandbox community with no real members.
  • 05liveProduction. The bot earned its way up.

One environment variable controls the entire ladder. If a bot misbehaves at 3 a.m., one Cronicle setting change kills it: no code change, no deploy, no vendor login required.

Even live ships with a paranoia cap. One post per run for the first twenty four hours.

The dial came from a real incident: a bot that passed test broke live in front of a real community in a subtle way. Every bot walks the ladder now.
Personal AI · The one that blocks

ContainmentGuard. The hook that denies the write.

Z1user-dataPAI/USER paths
Z2config-secrets.env, settings.json, .mcp.json
Z3runtime-memorybrain, memory, projects
Z4private-skillsunderscore-prefixed
Z5install-statesessions, history
Z6private-infrascratchpad, personas, logs

Pre-tool-use hook on Edit, Write, and MultiEdit. It scans content before it lands on disk. If the content matches an identity string or a credential prefix, the hook exits with code 2 and the write does not happen.

Eight identity patterns and six credential prefixes: sk-ant for Anthropic, eyJ for Supabase service-role keys, re_ for Resend, plus three more.

A policy that says "do not paste secrets" gets ignored on a tired Tuesday. A hook that physically cannot write the secret is on duty forever.

Ship the hook in log-only mode for 24 hours first. Watch what it would have blocked. Tune the patterns against real traffic. Then flip enforcement on.
Personal AI · Filter chain

Welcome-bot decisions encoded as seven independent filters.

  • 01Max age48 hours. Don't reply to two-week-old zombie threads.
  • 02Min age5 minutes. Don't be the first comment in seconds.
  • 03Max per run3 intros. Stop sudden floods.
  • 04Skip if20+ comments. The party already started.
  • 05Min body20 characters. Skips the wave-emoji "hi" posts.
  • 06ClassifierOn or off per community based on volume.
  • 07Coin flip85% post rate. A bot that comments on everything looks mechanical.

Each filter is testable in isolation, toggles per community, and writes its decision to Supabase with the chain version that processed it.

Seven filters in code. The chain is versioned, decisions land in Supabase, every filter is independently togglable per community.

Personal AI · The receipts

You cannot govern what you cannot query later.

  • 01CaptureHooks emit on every tool use, agent spawn, error, and cost event.
  • 02PersistSQLite + FTS5 + nomic embeddings. The sub-agent stop hook extracts findings from transcripts.
  • 03DecayEpisodic memories fade 2% per day. Semantic 0.5%. Below salience 0.1, prune.
  • 04QuerySearchable, citable. "What did the bot decide on May 10?"

One subtle thing worth holding. Rejections must distinguish transient from permanent. If a time-based filter says "this post is too young to comment on," that is transient. Mark it not-yet. Do not mark it seen. Otherwise the post becomes invisible forever when it matures.

Same engineering as an enterprise audit log at a fraction of the cost. The bridge to Part 2 is here: from local SQLite to vendor-scale audit infrastructure.
The bridge

The principle that holds on both sides.

Reduce AI as much as possible. Default to deterministic code.

Every regex you write is one prompt you do not. Every SQL is one drift surface you avoid. The LLM is for the ambiguity, not for the math, the comparisons, or the field extraction.

This rule scales differently at each end.

At a hundred calls a day, deterministic-first is a preference. You could get away with prompting everything. You would just be spending more.

At a hundred thousand calls a day, deterministic-first is a requirement. You cannot read the log. You cannot predict any single call. The only surface area you can govern with certainty is the part that does not run through the model.

The principle holds at both ends; the consequence of breaking it doesn't.

Deterministic first because deterministic code is the only thing auditable in the strong sense. LLM calls are probabilistic; auditing them is the whole rest of the talk.
Part 2 · Enterprise AI Quantum.

At a hundred thousand calls a day, every governance instinct from Part 1 stops working. You cannot enumerate, trace, or put a hook on every call and read what it did.

The substitute is statistical. Measure the distribution of outcomes instead of the events. Watch for drift in the prompt-and-model pair instead of waiting for crashes. Sample a small random subset every day and have a human read what the model decided.

The three shifts: distributions instead of events, drift instead of crashes, blind sampling instead of comprehensive audit. The next five slides are how the toolkit changes.
Enterprise · The lab

The Prompt Lab: a calibration set that non-engineers own.

Methodology

Subject matter experts review cases blind. The interface emulates the live workflow at submit time, so their call lands first. The AI's analysis and decision get revealed second, with the SME's commentary captured against it. Zero anchoring bias by construction.

The numbers, as of this week

528SME-reviewed cases in the golden set. 26eval runs to date. v4prompt in production. 74.7%agreement on the latest 186-sample subset.

The lab has three tabs: authoring, versioning, and head-to-head model comparison. Domain experts ship prompt patches without opening a pull request.

Engineers stop being the bottleneck for prompt iteration. SMEs ship the patches.

Enterprise · Sampled review

Blind human sampling, continuously.

Every day, a small random sample of yesterday's decisions gets queued for a human reviewer. Same blind UI as the SME calibration tool. The reviewer scores first. The AI's decision is revealed second. Disagreements feed the prompt patch queue.

Why dashboards alone fail:

Aggregate accuracy stays flat while a single category quietly collapses. You can lose ten points on a low-volume category and never see it in the headline number.

Confidence scores are vibes until a human disagrees. The model says 90% confident. The human says wrong. That disagreement is what calibrates the number; without it, 90% is just a number.

Tone, format, and framing drift never show up in metrics. They only show up in reads. The AI hedges too much, loses structure, or stops sounding like the brand. A reader catches that on day one.

The same blind UI runs SME calibration and daily blind review. The infrastructure compounds.

Five to ten samples a day takes a reviewer the time of a short coffee break. The catch rate is high because the dimensions they're judging are exactly the ones dashboards miss.
Enterprise · Failure mode

Set and forget is the failure mode.

Five real incidents from the last quarter, anonymized. Alerts caught none of them. Humans staying in the loop caught all of them.

  • 01Tone driftThe email composer prompt drifted across three macro styles over a month. Aggregate accuracy stayed fine. The reads sounded wrong.
  • 02Logging gapCo-pilot mode shipped without writing decision rows. 174 calls in. Zero log entries. Nobody noticed for ten days.
  • 03Vendor field bugIdentity vendor changed payload schema. The auto-invite pulled the wrong email field. Real users got nothing for two days.
  • 04Baseline driftThe manual review pipeline carried its own drift between SMEs and over time. The golden set carried that drift forward.
  • 05Gate blocking the gateThe criteria list that unlocks the next autonomy tier has been pending for weeks. The approval process itself became the blocker.
The system always tells you it's working. Only the reads tell you whether it is. Failure 5 is the meta one: even the approval pipeline can become its own bottleneck.
